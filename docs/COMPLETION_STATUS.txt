🔐 SECURITY NOTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• Tokens stored in localStorage (vulnerable to XSS)
  → Mitigated by HTML escaping on all user input
  → Consider moving to httpOnly cookies for production

• Password transmitted in plaintext over HTTP
  → MUST use HTTPS in production
  → Backend uses bcrypt for hashing

• Access tokens visible in Authorization header
  → Only sent over HTTPS
  → Token has short expiry (15 min default)

• Refresh tokens stored in DB as hashes (salted)
  → Cannot be used if DB is compromised
  → Revocation list prevents replay attacks
